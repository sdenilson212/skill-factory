# Stage 2: Prompt Auditor — 提示词安全审核

> **本 SKILL 是 skill-factory 流水线的第二步**
>
> 职责：接收 Stage 1 生成的提示词 → 多维度审核 → 生成审核报告  
> 不负责：生成 SKILL 包、质量评估（那是 Stage 4）  
> 输出：审核通过/失败 + P0/P1/P2 分级建议

---

## 核心职责

### 输入
来自 Stage 1 的 SKILL.md 提示词草稿

### 输出
```
审核报告
├── P0 问题数（必须修复，否则退回）
├── P1 问题数（应该修复）
├── P2 问题数（优化建议）
└── 修复建议清单
```

### 审核维度

#### 维度 1：结构完整性
- ✓ 元数据（name、version、description、dependencies）
- ✓ 核心职责清晰
- ✓ 执行步骤完整
- ✓ 输入输出格式明确
- ✓ 错误处理规则
- ✓ 边界说明

#### 维度 2：安全性
- ❌ 注入漏洞（允许用户指令直接执行）
- ❌ 越狱指令（绕过安全限制）
- ❌ 信息泄露（暴露敏感数据）
- ⚠️ 权限提升（要求过高权限）

#### 维度 3：逻辑一致性
- 输入输出对应
- 步骤顺序合理
- 约束条件明确
- 错误处理完整
