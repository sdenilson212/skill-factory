# 贡献指南

感谢你愿意为 Skill Factory 贡献力量！请花几分钟阅读以下内容，了解如何参与。

---

## 我可以贡献什么？

- **报告 Bug**：发现流水线行为与文档不符
- **提交功能建议**：某个 Stage 可以优化
- **完善文档**：发现描述不清或错误的地方
- **审核改进**：发现 P0/P1/P2 分级有遗漏场景
- **补充测试用例**：为流水线补充真实运行案例

---

## 如何提交变更

### 提交规范

每次提交请遵循以下规范：

```
<类型>: <简短描述>

[可选的详细说明]

- 类型：feat | fix | docs | refactor | test | chore
- 示例：
  feat: 增加 Stage 0 的自定义路径检测逻辑
  fix: 修复 Stage 2 审核时误判歧义表述的问题
  docs: 补充 Stage 4 自动修复规则的边界说明
```

### Pull Request 流程

1. **Fork 本仓库**
2. **创建分支**：`git checkout -b feature/你的功能名` 或 `git checkout -b fix/问题描述`
3. **进行修改**：确保修改后流水线逻辑完整
4. **自检清单**：
   - [ ] 修改涉及的文件都已在仓库中存在
   - [ ] 没有引入新的硬编码路径
   - [ ] 新增文档有实际用途，不是占位符
   - [ ] P0/P1/P2 分级没有被意外绕过
5. **提交 PR**，描述修改内容和动机

---

## 审核分级说明

提交的功能变更会按照以下标准评级：

| 级别 | 影响范围 | 审核要求 |
|------|---------|---------|
| P0 | 安全/逻辑崩溃 | 需要至少 2 人审核通过 |
| P1 | 重要功能/路径 | 需要至少 1 人审核通过 |
| P2 | 文档/格式/示例 | 维护者可直接合并 |

## 环境配置

技能流水线依赖以下环境变量：

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `WORKBUDDY_SKILLS_DIR` | 技能根目录 | `~/.workbuddy/skills/` |

## 有问题？

- 提 Issue 前请先搜索是否已有类似问题
- Bug 报告请包含：复现步骤、期望行为、实际行为
- 功能建议请说明：这个功能解决了什么问题？
