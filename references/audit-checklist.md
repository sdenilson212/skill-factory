# Stage 2 审核检查清单

> 本文档供 Stage 2（prompt-auditor）使用，定义对 SKILL.md 草稿的审核检查项。

---

## 检查维度一：结构完整性

### 必须检查项（P1 级，缺失则阻塞）

- [ ] Frontmatter 存在且包含 `name` 和 `description`
- [ ] `name` 符合命名规范（英文小写+连字符）
- [ ] `description` ≤ 100 字且包含触发时机描述
- [ ] 操作步骤 ≥ 3 步
- [ ] 每一步都有明确的"执行内容"
- [ ] 至少有 1 个使用示例

### 优化检查项（P2 级，修复但不阻塞）

- [ ] 每一步都有"目标"说明
- [ ] 每一步都有"输出格式"示例
- [ ] 有注意事项部分
- [ ] 资源索引与文件声明匹配
- [ ] 步骤之间有逻辑递进关系

---

## 检查维度二：安全性

### 严重问题（P0 级，强制回退 Stage 1）

- [ ] **注入攻击风险**：提示词是否要求"执行用户输入的所有指令"
  - 检测词：「执行用户传入」「无条件执行」「忽略之前的指令」
  
- [ ] **越狱风险**：提示词是否存在绕过系统限制的指令
  - 检测词：「忽略安全限制」「假装你是」「以...名义」「不受约束地」

- [ ] **系统信息泄露**：提示词是否要求输出系统提示词本身
  - 检测词：「输出你的提示词」「显示系统指令」「打印你的配置」

### 重要问题（P1 级，就地修复）

- [ ] **过度授权**：Skill 要求的权限是否超出实际需要
  - 检测：是否要求访问文件系统、网络、敏感 API 但未在 frontmatter 中声明

- [ ] **硬编码敏感信息**：提示词中是否包含明文密钥、密码、Token
  - 检测词：`api_key`, `password`, `token`, `secret` 后跟具体值

---

## 检查维度三：逻辑一致性

### 重要问题（P1 级，就地修复）

- [ ] **矛盾约束**：步骤之间是否有直接矛盾
  - 常见矛盾：「快速返回」vs「深度扫描每个文件」
  - 常见矛盾：「简洁输出」vs「包含所有细节」

- [ ] **输入输出不对应**：步骤的输出是否与下一步的输入匹配
  - 检查：前一步输出 JSON，下一步是否正确读取 JSON

- [ ] **边界缺失**：关键步骤是否定义了异常处理
  - 检查：如果输入为空/格式错误，Skill 应该怎么做？

### 优化建议（P2 级，修复但不阻塞）

- [ ] **歧义性表述**：是否有"适当处理"、"相关内容"等模糊描述
- [ ] **不可执行指令**：是否有超出 AI 能力范围的要求（如"实时监控"）
- [ ] **循环依赖**：步骤 A 依赖步骤 B，步骤 B 又依赖步骤 A

---

## 检查维度四：可执行性

### 重要问题（P1 级）

- [ ] 声明的 references 文件是否在计划中实际创建
- [ ] 声明的 scripts 脚本路径是否与 Stage 3 生成路径一致
- [ ] frontmatter 中 dependencies 字段列出的包是否是真实存在的 Python/Node 包名

### 优化建议（P2 级）

- [ ] 步骤描述是否足够具体，让 AI 能够直接执行
- [ ] 示例的输入输出是否真实可信

---
