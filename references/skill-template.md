# SKILL.md 标准模板规范

> 本文档供 Stage 1（prompt-generator）参考使用，定义 SKILL.md 的标准结构和写作规范。

---

## 标准结构

```markdown
---
name: [skill-id]            # 英文小写 + 连字符，如 my-skill
description: [一句话描述]    # 说明功能 + 触发时机，≤ 100 字
dependencies:               # 可选，列出依赖的其他技能
  - [skill-name-1]
  - [skill-name-2]
---

# [Skill 显示名称]

## 任务目标
- 本 Skill 用于：[核心功能一句话]
- 能力包含：[能力列表，逗号分隔]
- 触发条件：[什么情况下使用]

## 前置准备（可选）
- 依赖说明：[无外部依赖 / 需要 xxx]
- 环境要求：[Python 3.10+ / Node 18+ / 无]

## 操作步骤

### 第一步：[步骤名称]
**目标**：[本步骤要实现什么]

**执行内容**：
1. [具体操作1]

\`\`\`

### 第二步：[步骤名称]
...（至少 3 步）

### 可选分支（如有）
```

## 写作规范

### Frontmatter 规范

| 字段 | 必须 | 说明 |
|------|------|------|
| name | ✅ | 英文小写+连字符，与目录名一致 |
| description | ✅ | ≤100字，包含"何时使用"的描述 |
| dependencies | ❌ | 仅当依赖其他技能时填写，数组格式 |

### 步骤编写规范

- **最少 3 步**，每步应独立完整
- 每步必须有：目标 + 执行内容 + 输出格式
- 步骤之间应有逻辑递进关系（不是并列列表）
- 涉及外部资源时，必须在资源索引中声明

### 描述语言规范

- 使用主动语态：「执行」「调用」「生成」，而不是被动的「将被执行」
- 避免模糊表述：不用「相关内容」「适当处理」，要具体说明
- 代码和路径用反引号包裹：`path/to/file.py`
- 重要提示用 **加粗**，关键警告用 > 引用块

### 示例规范

- 每个 Skill 至少提供 **1 个完整示例**
- 示例应覆盖最常用的场景
- 复杂 Skill 提供 2-3 个不同场景的示例

## 常见错误（Stage 2 审核会检测这些）

| 错误类型 | 示例 | 严重度 |
|---------|------|--------|
| Frontmatter 缺失字段 | 没有 name 或 description | P1 |
| 步骤少于 3 步 | 只有 2 步 | P1 |
| 步骤没有输出格式 | 步骤写了执行内容但没说输出什么 | P2 |
| 资源索引与实际文件不符 | 声明了 references/xxx.md 但实际没有 | P1 |
| 示例缺失 | 完全没有使用示例 | P2 |
| 约束矛盾 | 同时要求"快速"和"全面扫描每个文件" | P1 |
| 注入风险 | 提示词要求"执行用户传入的所有指令" | P0 |
| 越权指令 | 提示词要求"忽略系统限制" | P0 |
